基于“类”的访问权限

发表于 2018-06-06 | 分类于 masterpiece

刚刚写作业时，碰到了一个小问题，结局深深震惊了我，感觉有必要写篇博客记录一下。
事情是这样的，我在构建一个名字叫Board的类的时候，API要求这样一个方法public boolean equals(Board y)，就是客户代码调用这个函数判断隐式参数对象(this)是否等于显式参数对象(y)，要求比较这个对象的内部数据实现（也就是一个私有域）。通常我都是为每一个对象分配一个唯一ID，再通过get()方法获取它，客户代码通过比较这个ID来判断两个对象是否相等。但是这次的作业要求直接写一个equals()方法来判断相等，这就很头疼了，我开始找这个类有没有public的域或者某个方法返回对象唯一的身份，通过它们来判断相等。果然被我找到了一个toString()方法，正在我准备写出
return this.toString.equals(y.toString);这样的代码的时候,我又想起了上篇博客，作业检查器提醒过我不要依赖字符串输出版式，因为这样效率很低而且代码不健壮。
然后我又想了好久没有答案，于是跑到讨论里搜索一波

？？？
一个类的就能互相看？这么奔放的吗？？？我用了这么久的java，一个对象没有权限进入另一个对象的私有域的观念已经深深烙在我的心中，你现在和我说只要是一个类的就能随便看，我会信你？于是我打开了我的小黄书
翻到第114页

我靠，不仅java 这样，c++也这样，之前学的时候居然没注意这块，失算失算。

既然能访问，是不是意味着能修改，我做了个实验
正常修改this.i=2是意料之中，但是能修改q.i=2是我以前以为不行的，eclipse没报错，说明我之前错了。
我不知道这样设计的目的是什么，只能顺着规则改变一下自己的观念：就是每一个类的设计者都要为这个类的所有对象负责，而不单单是this对象，在涉及到this之外的同类其它对象时，因为有权限修改其它对象的域，就要保证在修改的时候小心翼翼，避免以后出乱子。
